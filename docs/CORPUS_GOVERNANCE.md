# DeepLaw 语料治理与发布政策

## 目的与适用范围

DeepLaw 是跨 Agent 使用的公共中国法律研究底座。它提供的是可定位、可校验的法律
研究材料，不替代法律效力审查、案件事实证明、法律适用判断或人工复核。

本政策适用于：

- 官方 DOCX、PDF、OFD 等原始法源；
- 从原件提取的规范化文本、结构和位置；
- OCR、MinerU、标签、图谱、向量、摘要和 Wiki 等派生数据；
- SQLite release、搜索索引、证据卡和 release 元数据；
- 本地构建、分发、激活、撤销和更新流程。

案件证据、案件聊天、工作笔记、身份信息、交易数据和案件项目私有文档不属于
DeepLaw 公共语料，任何情况下都不得进入公共 release、缓存、日志、评测集或派生 Wiki。

## 权威层级

DeepLaw 按以下顺序判断资料权威性：

1. 发布机关或经核验的官方托管站点提供的原始文件；
2. 与原件 SHA-256、来源 URL 和定位信息绑定的不可变 release；
3. 从原件确定性提取、且可回链到页码或段落的规范化文本；
4. OCR、结构推断、标签、检索排名、图谱、向量、摘要和 Wiki 等派生数据。

第 4 层永远不能覆盖前三层，也不能单独证明法律效力。GitHub 镜像、商业数据库截图、
新闻稿、论坛转载、模型记忆和未核验网页正文不能成为 DeepLaw 法律真源。

## 角色和信任边界

- **采集者**：从官方入口取得候选文件并记录来源，不决定其法律效力。
- **构建器**：校验文件并生成候选 release；构建成功不等于可以公开或激活。
- **语料复核者**：复核题名、文号、发布机关、版本、结构、OCR 和来源。
- **法律版本复核者**：确认公布、施行、修改、废止、替代和历史适用关系。
- **许可与隐私复核者**：决定二进制原件和派生文本能否分发，并复核案例再识别风险。
- **发布者**：在所有适用闸门通过后签发和激活 release。
- **运行时**：只能读取已批准 release，不能采集、修改、删除或发布语料。

MCP 运行时不得拥有构建和发布权限。`build`、OCR、人工复核和激活属于离线管理路径，
不能作为 Agent 可调用工具暴露。

## Release 生命周期

每个 release 必须处于下列状态之一：

```text
candidate -> source_verified -> content_reviewed -> approved -> active
                                             \-> rejected
active -> superseded | revoked
```

- `candidate`：仅完成初步收集，不能提供给生产 Agent。
- `source_verified`：路径、格式、大小、哈希和官方来源通过机械校验。
- `content_reviewed`：提取、结构、OCR、版本、隐私和许可审查完成。
- `approved`：发布清单签发，可以进入受控分发。
- `active`：某一部署当前默认使用，但仍可按 `release_id` 读取历史版本。
- `superseded`：被新 release 替代，保留用于历史复核，不静默覆盖。
- `revoked`：发现完整性、来源、许可或重大内容错误；停止新查询并保留审计记录。
- `rejected`：未通过发布闸门，不能通过 `--allow-needs-ocr` 等开发选项绕过。

当前 v0.1 构建器会校验路径、大小、SHA-256、HTTPS 来源和 PDF 文本质量，但尚未机械
执行本政策的全部人工闸门。`deeplaw build --activate` 只改变本地 ACTIVE 指针，不是
许可、隐私、时效或法律有效性证明。生产发布流程必须在其外层实施本政策。

## 发布闸门

### G0：输入隔离

- 输入必须是显式 allowlist 中的候选包和 manifest；不得扫描整个 Downloads、Obsidian
  Vault、工作区或案件目录。
- 拒绝符号链接、路径穿越、特殊文件和 manifest 未登记的正文。
- `.DS_Store`、临时文件、隐藏索引、聊天记录和操作说明不得进入正文索引。
- 候选输入目录只读；DeepLaw 在独立输出目录生成 release。

### G1：来源和完整性

每份文件至少具备：

- 稳定相对路径和唯一文档标识；
- 官方 HTTPS 来源 URL、发布机关和托管机关；
- 原始字节数和完整 SHA-256；
- 获取日期和复核日期；
- 文件题名、格式和必要的文号或案例编号。

路径、字节数、hash、MIME、文件数或来源任一不一致时，整个候选 release 失败关闭。
同 hash 重复文件必须显式解释，不能被静默去重后继续发布。

### G2：文件安全与提取质量

- DOCX 必须通过 OOXML 容器校验，并检查宏、外部关系和嵌入对象。
- PDF 必须记录是否加密、是否含 JavaScript/附件、页数、文本覆盖率和异常字符。
- 有可靠文本层的 PDF 优先原生提取，不得因统一流程而强制 OCR。
- 扫描件或低质量文本页只能进入 `needs_ocr`；完成 OCR 和人工抽查前不得作为完整正文。
- OCR/MinerU 结果是派生物，必须记录工具、版本、配置和结果 hash，并保留后端能够提供的
  页码、坐标和置信度；后端不能提供的定位或置信度字段必须显式记录为缺失并进入人工
  复核，不能用空值暗示已经验证。
- 原件永不被 OCR 或规范化输出覆盖。

当前 `deeplaw.sqlite/v3` 新构建会记录 Tesseract 与 `pdftoppm` 的解析版本、
`dpi=300`、`language=chi_sim+eng`、`psm=3`、逐页文本和 extracted-text hash；当前
Tesseract 路径仍不保存词级坐标或置信度。MinerU 路径要求操作员预装模型并设置
`MINERU_MODEL_SOURCE=local`，固定使用本地 `pipeline` backend；`content_list.json` 可保留
页码，Markdown fallback 不能补造坐标或置信度。两条 MinerU 输出路径都会记录已解析的
MinerU 版本。这些记录提高可重建性，但不代替逐页人工抽查，也不会追写到历史 release。
在缺失定位/置信度字段和相应人工复核关闭前，这些 OCR 路径仍未完整通过 G2。

### G3：结构和引文可复核性

- 保留原始文档顺序、章节路径、条款编号、段落或页码位置和脚注。
- 每个 segment 绑定原件 hash、segment hash、解析器版本和稳定 locator。
- Word 缓存的页数、字数和 Heading 样式不能在未验证时作为结构真相。
- 法条拆分、跨页合并和脚注归属必须通过确定性测试及人工抽样。
- 任何证据卡都必须能通过 `segment_id` 获取原文并通过 receipt 校验。

### G4：法律版本和时效

每份规范性文件必须结构化记录：

- 文件类型、发布机关、文号和法域；
- 公布日期、施行起止日期和效力状态；
- 修改、废止、替代、配套和解释关系；
- 状态结论的官方依据、复核者和复核时间。

缺少数据时使用 `unknown` 或 `requires_human_review`，不得猜测为 `current`。修正案和
修改决定不能由 LLM 自动合并为现行文本；如生成合并视图，必须保存确定性变换记录、
逐条来源和人工批准。日期过滤只说明候选时间相符，不自动证明案件适用。

### G5：隐私与案例治理

- 公共法源服务不得接收或保存案件私有材料。
- 案例即使使用化名，也要检查残留身份、联系方式、账号、地址、设备标识和可再识别事实。
- 扫描文件必须在 OCR 后重新执行 PII 检查。
- 自动扫描只能作为初筛；案例正文公开需要人工复核和来源许可结论。
- 原件中的作者、最后修改者、本机路径等非必要元数据不得出现在公开 API 和派生文本中。
- 原件为保持 hash 不得就地清洗；公开派生层通过白名单字段投影去除非必要元数据。

### G6：许可和再分发

“官方网站可下载”不自动等于“允许重新打包公开发布”。每份来源至少记录：

- 作品/文件类别；
- 来源站点条款及复核时间；
- 原始二进制、提取文本和短引文分别允许的分发范围；
- 第三方版式、标识、案例汇编或数据库权益风险；
- 许可结论、依据和复核者。

许可未明确时，公开仓库只发布代码、schema、来源 URL、hash 和本地构建流程；不提交
原始 DOCX/PDF，不发布可还原完整正文的数据库。DeepLaw 自身的代码许可证不改变法源
文件和上游组件的许可条件。

### G7：检索与输出质量

- 精确题名、文号、条款和版本查询必须优先于宽泛语义召回。
- 搜索最多返回五张证据卡，摘要总字符数遵守公开 contract 上限。
- `get` 只能按精确 `segment_id` 读取；不得用一次宽泛查询获取整库正文。
- 无命中只表示本次 release 和查询范围内无命中，不表示法律不存在或行为不成立。
- 向量、图谱、reranker 和 Wiki 上线前必须在版本、引文、无命中和泛词基准中证明增益。
- 检索评测不得包含案件私有材料，也不得把模型生成答案当作金标准。

### G8：Release 完整性和激活

- release 目录、SQLite 数据库和 release manifest 均不可变；更新创建新 `release_id`。
- release manifest 记录 source manifest hash、数据库 hash、文档/segment 数、审核状态和
  派生能力开关。
- 激活前重新计算数据库 hash，验证 schema 和抽样 receipt。
- 一次 Agent turn 或正式报告固定一个 release；中途更新不能改变正在使用的语料。
- 当前 stdio MCP 进程在 lifespan 启动时解析并校验一次 `ACTIVE` 或 `DEEPLAW_DB`，随后在
  整个进程生命周期固定并复用该只读 release；切换 `ACTIVE` 只影响新进程，不能热切换
  已运行进程。
- 回滚通过切换到既有已批准 release 完成，不修改旧 release。

## 只读运行时要求

- SQLite 以只读、immutable 模式打开；低层 MCP server 只通过本地 stdio 提供
  `search`、`get`、`verify` 和 `release_info`，不监听 HTTP。
- 不提供 `add`、`update`、`remember`、`feedback`、`wiki_write` 或后台学习工具。
- 查询、模型输出和用户反馈不能写回公共 corpus。
- 运行时不得下载模型、抓取网页或在校验失败时回退到模型记忆。
- 法律 release 不得直接打开案件 SQLite/DuckDB，也不得接受案件路径、`caseId` 或完整事实。

MinerU 和 Tesseract/Poppler 只属于显式选择的离线构建路径，不是 MCP 运行时依赖。要求
`MINERU_MODEL_SOURCE=local` 会阻止 DeepLaw adapter 使用 MinerU 的正常模型下载路径，但
不是操作系统级网络沙箱；需要更强隔离时由构建环境另行实施。

## 派生数据政策

以下内容必须标为 `derived`，可删除、可重建且无独立证据权：

- OCR 和 MinerU 文本；
- 分词、FTS 辅助 token、embedding 和向量索引；
- LLM 标签、实体、关系、专题摘要和 Wiki；
- 查询改写、reranker 分数和答案草稿。

派生记录至少绑定输入 release、来源 ID、生成工具/模型、版本、时间、配置和输出 hash；
页码、坐标或置信度不可用时，还必须记录这一限制及相应人工处置。
删除派生层后，原始 release 的精确检索、读取和 receipt 校验仍应可运行。

## 更新、撤销和事故处理

新增法源或版本修正必须创建新候选 release，重新通过全部适用闸门。发现下列问题时应
立即停止激活或撤销 release：

- 官方来源、hash 或 locator 无法验证；
- 法条错位、OCR 关键字错误或漏页；
- 效力状态、版本关系或发布机关错误；
- 未授权再分发或隐私泄漏；
- 跨 release receipt、数据库被修改或查询返回不属于当前 release 的 segment。

撤销记录应包含受影响 release、原因、时间、发现方式、替代 release 和复核结果，但不能
写入案件材料或敏感正文。客户端必须显式显示 release 不可用，不能静默切换到网络搜索或
模型记忆后继续声称是现行法。

## 禁止事项

- 把公共法源与案件私有库合并到同一数据库或全局向量库；
- 用 `case_id` filter 代替物理隔离；
- 把 Obsidian Vault、Downloads 或整个工作区作为自动摄取根目录；
- 覆盖、修补或重新保存官方原件后仍沿用旧 hash；
- 让 LLM 决定法条效力、修订关系或直接改写真源；
- 把 OCR、Wiki、相似案例或检索分数当作法律权威；
- 因构建成功而跳过许可、隐私和时效复核；
- 在公开仓库、日志、fixture 或错误信息中提交原始法源、私有材料或本机用户路径。

## 最低审计记录

每个已批准 release 应保留：

- 输入 manifest 及其 SHA-256；
- 全部来源 URL、文件 hash、大小、获取时间和验证结果；
- 解析器/OCR 及渲染器版本、配置、派生文本 hash、构建报告、不可用的坐标/置信度字段、
  警告和人工处置；
- 法律时效、许可和隐私复核记录；
- 数据库 hash、release manifest、评测结果和批准记录；
- 激活、替代、回滚和撤销历史。

这些记录证明来源和过程，不证明案件事实，也不自动证明特定案件的法律适用。
